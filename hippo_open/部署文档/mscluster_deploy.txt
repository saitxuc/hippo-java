1.修改hippo/hippo.run/conf/hippo-single.properties

涉及的通用项修改如下:
broker.name = {broker_name}
broker.uris = hippo://{hostip}:61000
broker.type = mscluster
cluster.name = {mscluster_name}
zk.url = {zookeeper}
jmx.connector.host = {hostip}

mdb引擎对应的配置项:
一定要配置该项
broker.store = mdb   

#mdb store engine config使用内存的空间大小
db.limit = {capacity}  例:5g 

该机器的桶号, 可以写多个或者单个,逗号分割 例子 0,1,2
db.buckets = 0

过期处理配置项目,每个不同的分片size,处理验证的数据数量
store.expire.count.limit = 512

lru配置项,数据满了以后,将会根据每个分片的过期时间排列,取出前30%的数据空间作为lru使用的空间
store.lru.fate = 0.3


使用levelDB的配置项
一定要配置该项
broker.store = levelDb  

db.limit = {capacity} 例子:100g  使用硬盘的空间大小

是否使用mdb引擎作为levelDB的缓存,可关闭,关闭后leveldb.mdb.use.limit就不起作用了
leveldb.mdb.use.flag = true  

mdb引擎的缓存大小
leveldb.mdb.use.limit = {mdbcapacity}  例子2g


2.修改hippo/hippo.run/bin/hippo
-XX:MaxDirectMemorySize=?g  建议值为mdb的使用值加上分配netty的1G使用堆外空间


在主备的模式下使用hippo,一定要在配置项中指定对应的引擎,
db.limit,db.buckets,broker.name,cluster.name,zk.url 这4个选项保持主备机的一致
